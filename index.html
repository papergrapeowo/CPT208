Literature Review

1
XR Exhibit+

XR Exhibit+: An AI-Enhanced XR Museum Guide

Strengths
  • Implements core 3D/XR visualization functions as a mature reference.
  • Supports semantic-based exhibition knowledge association via graphs.
Limitations
  • Relies on dedicated XR headsets; low accessibility for mobile users.
  • Weak AR effects on mobile phones for fine relic details.
  • Difficult to support full-museum services due to limited exhibits.
Implication: Draw on 3D visualization to optimize modeling; focus on mini-program deployment and abandon headset dependence.
2
OFA-Based VQA
A cultural relic visual question answering system based on multimodal models.
Click to explore details →

Intelligent Museum Guide System Based on OFA: Cultural Relic Visual Q&A

Strengths
  • Supports natural AI Q&A via exhibit photo-taking.
  • Processes multimodal info (image + text) for natural interaction logic.
  • Real-time response based on user interests improves visiting freedom.
Limitations
  • Lacks 3D modeling and immersive display; single experience dimension.
  • Highly dependent on recognition accuracy (affected by lighting/angles).
  • No personalized path adaptation or complete guided tour closed loop.
Implication: Learn from its VQA logic to optimize our Q&A module; use 3D modeling to compensate for visual lack; combine Q&A with artifact self-narration.
3
AI Indoor Experience
An immersive system based on 3D reconstruction, AI narration and indoor positioning.
Click to explore details →

Design of Indoor Interactive Experience Based on AI Technology

Strengths
  • Includes a complete technical path for 3D modeling and scene rendering.
  • Multimodal design (voice/touch) improves system universality.
  • Real landing cases provide a practical reference framework.
Limitations
  • Lacks free AI Q&A functions with weak interactivity for open questions.
  • High device performance requirements; difficult to run on ordinary phones.
  • Complex interaction process results in high learning costs for users.
Implication: Refer to its 3D modeling process while simplifying interaction for mini-programs; add free AI Q&A to enhance flexibility.
4
Natural Interaction & LLMs
Enhancing user experience in virtual heritage through LLM-driven natural interaction.
Click to explore details →

Natural Interaction in Virtual Heritage: Enhancing UX with LLMs

Strengths
  • Implements smooth conversational interaction for free-form exhibit questioning.
  • Optimized through two rounds of user research to ensure high satisfaction.
  • Supports voice interaction, reducing input thresholds for mobile scenarios.
Limitations
  • Developed for VR environments; lacks direct mobile accessibility.
  • Limited support for 3D artifact browsing and detailed physical observation.
  • Challenges in dialogue response latency and accuracy in large-scale scenarios.
Implication: Use its conversational design to optimize AI Q&A but focus on mobile mini-program deployment and 3D modeling.

04Ideation: Iterative Evolution

From initial messy thoughts to a structured UI. We navigated through four major versions before reaching the final concept.

A simple list-based interface. Rejected for being too static and failing to provide the immersive interactive experience visitors expected.

Rejected

Attempted a map-driven interface to help orientation. Ruled out as the complex map elements distracted users from the artifact details.

Rejected

The AI Framework

Introduced 3D viewing and AI chat. However, the interface felt cluttered, and the AI's tone was criticized for being too academic and robotic.

y4: Final Selected

Personified Digital Twin

  • Bottom Layer: EN/CN toggle for global accessibility.
  • Core: Immersive 360° artifact interaction.

05Version 1 Design

The initial high-fidelity prototype focuses on establishing the core visual language and testing the 3D-AI interaction loop.

01. Discovery & Cultural Context

The home screen establishes an immediate emotional connection by showcasing the museum's iconic architecture. It provides essential historical background, lowering the cognitive barrier for first-time visitors.

  • Visual Gateway: Immersive header photography to enhance aesthetic appreciation.
  • Clear Navigation: Intuitive bottom tabs for seamless switching between exploration and interaction.

02. Curated Artifact Browsing

This "Exquisite Artifacts" list allows users to scan through high-value relics. Each card provides a visual preview and key metadata (Dynasty and Origin) for rapid information intake.

  • Information Hierarchy: Clear categorization to help users prioritize their visit path.
  • Consistency: Standardized card layouts to reduce visual fatigue during browsing.

03. Interactive AI Dialogue

The core interactive feature gives artifacts a "voice" through LLMs. Instead of reading rigid labels, users engage in personified conversations with relics like the Pearl Sarira Stupa.

  • Personified Persona: Artifacts introduce themselves to foster empathy and curiosity.
  • Personalized Q&A: Real-time responses to natural language questions, turning visits into active learning.

Design Improvements

Increased font sizes by 20% for better readability among elderly users with vision impairments.

✓ UI Optimization

Refined visual design with improved layout hierarchy and cleaner interface elements for enhanced user experience.

✓ Bilingual Toggle (EN/CN)

Implemented Chinese/English language toggle to support international visitors and enhance accessibility for non-native speakers.

✓ AI Conversation History

Added chat history saving feature, allowing users to review and continue previous AI conversations anytime.

Final Design Showcase

01. Bilingual Translator

Instant translation between Chinese and English helps international visitors understand exhibits.

  • Real-time Translation: Quick language switching for all content.
  • Contextual Understanding: Accurate translation of cultural terms.

02. AI-Powered Chat Assistant

Natural language conversation with our AI guide answers personalized questions about exhibits.

  • Instant Answers: Real-time responses to visitor inquiries.
  • Context-Aware: Remembers conversation history for continuous dialogue.

03. Quiz Game

Interactive quiz challenges test your knowledge about exhibits with engaging gameplay.

  • Knowledge Check: Fun quizzes to reinforce learning.
  • Progressive Difficulty: Questions adapt to your knowledge level.

04. Jigsaw Puzzle

Interactive jigsaw puzzle featuring museum artifacts for an immersive learning experience.

  • Hands-on Learning: Engage with artifacts through play.
  • Multiple Levels: Different difficulty levels for all ages.

05. 3D Artifact Display

High-quality 3D rendering of museum artifacts with detailed inspection capabilities.

  • Rotatable View: 360-degree artifact rotation.
  • Detail Exploration: Zoom in to examine intricate craftsmanship.

🤖 AI Usage Reflection

AI Citation: Gemini 1.5 Pro, v1.0, accessed 2026-04-15. https://gemini.google.com/. Used for system architecture generation, 3D rendering logic optimization, and responsive code structure.
Case 1: System Architecture Generation

- P (Prompt): "Design a technical architecture diagram for an intelligent museum exhibit system featuring a React frontend, Three.js 3D viewport, and an LLM-based personified dialogue system."

- O (Outcome): Gemini produced a clear hierarchical structure but missed the data persistence layer for storing chat histories.

- A (Action): We manually integrated the Vercel PostgreSQL layer into the Gemini-generated logic to ensure a complete data loop.

Case 2: Technical Interaction Logic

- P (Prompt): "Write a React hook to handle 3D model auto-rotation and responsive canvas resizing for a mobile-first museum exhibit."

- O (Outcome): The AI-generated code was technically sound but caused unexpected layout shifts on smaller screens.

- A (Action): We modified the CSS flexbox properties and optimized the debouncing logic in the resize handler to stabilize the mobile UI.
